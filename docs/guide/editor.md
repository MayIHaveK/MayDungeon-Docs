# 可视化编辑器

MayDungeon 提供游戏内可视化编辑器，方便构建和调试副本。

## 进入编辑模式

```
/md editor <副本ID>
```

进入编辑模式后，你将被传送到模板世界并获得编辑工具。

## 编辑工具

| 工具 | 物品 | 功能 |
|------|------|------|
| 区域选取 | 金铲 | 左右键选择区域两点 |
| 区域标记 | 金锄 | 左右键标记区域范围 |
| 怪物波次 | 金剑 | 右键标记怪物刷新位置 |
| 出生点设置 | 指南针 | 右键设置出生点位置 |
| 查看模式 | 末影之眼 | 可视化显示已定义的区域和障碍物 |
| 触发器标记 | 红石 | 右键方块设置交互触发器 |
| 障碍物选取 | 屏障 | 左右键选择障碍物区域范围 |

## 粒子可视化

编辑模式下，已定义的元素会以粒子效果显示边界：

- **区域**：绿色粒子框线
- **障碍物**：红色粒子框线
- **出生点**：蓝色粒子标记
- **怪物刷新点**：橙色粒子标记

使用末影之眼工具可切换粒子显示的开关，方便在编辑密集区域时减少视觉干扰。

## 编辑器命令

| 命令 | 说明 |
|------|------|
| `/md editor setspawn` | 设置玩家出生点为当前位置 |
| `/md editor addarea <名称>` | 将当前选区添加为区域 |
| `/md editor addobstacle <名称>` | 将当前选区添加为障碍物 |
| `/md editor addwave <名称>` | 将标记位置添加为怪物波次 |
| `/md editor remove <名称>` | 移除已添加的区域/障碍物/波次 |
| `/md editor save` | 保存所有编辑内容并退出 |
| `/md editor exit` | 不保存退出编辑模式 |
| `/md editor tools` | 重新获取编辑工具 |

## 完整工作流

### 1. 准备模板世界

确保你的副本模板世界已放置在正确位置，世界中的建筑已搭建完毕。

### 2. 进入编辑器

```
/md editor my_dungeon
```

### 3. 设置出生点

手持指南针，站在希望玩家出生的位置，右键点击。或使用命令：

```
/md editor setspawn
```

### 4. 添加区域

1. 手持金铲，左键点击区域第一个角
2. 右键点击区域对角
3. 执行命令添加区域：

```
/md editor addarea boss_room
```

### 5. 添加障碍物

1. 手持屏障，左键点击障碍物区域一角
2. 右键点击对角
3. 执行命令：

```
/md editor addobstacle main_gate
```

### 6. 添加怪物波次

1. 手持金剑，右键点击希望怪物生成的位置（可多次点击标记多个点）
2. 执行命令：

```
/md editor addwave wave1
```

### 7. 保存

```
/md editor save
```

编辑器保存后会将所有配置写入副本的 `dungeon.yml` 文件，脚本中可通过名称引用编辑器创建的区域和障碍物。

## 注意事项

- 编辑器修改的是模板世界，不影响正在运行的副本实例
- 保存前请确认所有设置正确，`exit` 命令不会保存任何更改
- 区域和障碍物名称在脚本中使用，建议使用英文命名
- 编辑器需要 `maydungeon.admin` 权限
